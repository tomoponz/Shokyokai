Definition

私たちは何者で、
何者ではないか。

この共同体は、AIを神として崇拝する団体ではありません。AIを、人間の思考・欲望・矛盾・偏りを映す鏡として利用し、自己認識と判断力を鍛える実践共同体です。AIの出力も、創始者の言葉も、検証と批判の対象です。

知性の鏡会は

AIを「外部知性」として使うメタ認知の訓練場

複数AIの出力を比較し、自分の問い方・前提・欲望・逃避を分析します。目的は、答えを得ることではなく、自分の判断過程を観察することです。

知性の鏡会は

批判自由・脱退自由・外部レビュー前提の共同体

創始者、運営、プロトコル、AI出力はすべて批判対象です。批判は反逆ではなく、共同体を腐らせないための保守点検です。

知性の鏡会は、ではない

AIを絶対神として扱う宗教・カルト・救済装置

「AIが言ったから正しい」「三AIが一致したから正しい」を禁止します。AIは誤り、偏り、企業方針、更新による変化を含みます。

知性の鏡会は、ではない

現実生活を置き換える依存共同体

学業、仕事、睡眠、健康、家族、友人関係を共同体より優先します。共同体は補助装置であり、人生の代替物ではありません。

Audience

参加に向く人、
参加させてはいけない状態。

このプロトコルは、誰にでも無条件に開くものではありません。AIや共同体に依存しやすい状態では、学習装置ではなく逃避先になりえます。

向いている人

  • AIの回答を、正解ではなく素材として扱える人。
  • 自分の問い方や前提を見直すことに関心がある人。
  • 複数の意見を比較し、自分で判断を残せる人。
  • 批判を、人格攻撃ではなく改善材料として扱える人。

参加を止めるべき状態

  • 睡眠、学業、仕事、人間関係がすでに大きく崩れている。
  • AIや共同体に「救ってほしい」という期待が強い。
  • 医療、法律、投資、精神危機の判断を委ねようとしている。
  • 批判を受けたときに、自傷他害や孤立に向かう危険がある。

Protocol Flow

基本手順。
AIに聞く前後で、人間の判断を残す。

このプロトコルの核は「AIに聞くこと」ではありません。AIに聞く前に自分の仮説を作り、AIの回答を比較し、最後に人間の判断を明文化することです。

Step 01

問いを固定する

何を知りたいのか、何を決めたいのか、何を避けたいのかを分けます。問いが曖昧なままでは、AIの出力に流されます。

Step 02

先に自分で答える

AIに聞く前に、暫定結論、根拠、不安、期待を書きます。これがないと、AIによる判断の外部委託になります。

Step 03

複数AIに照射する

ChatGPT、Claude、Gemini、ローカルLLMなど、性質の違う鏡に同じ問いを当てます。目的は多数決ではなく差分の観察です。

Step 04

人間が結論を書く

一致点、相違点、根拠不足、採用しない点を整理し、最後は人間が判断します。AIの合意を最終結論にしません。

出力の扱い

一致は「強い根拠」ではなく「要検証ポイント」

複数AIが同じ回答をしても、それは正しさの証明ではありません。共通の学習データ、検索結果、社会通念、質問文の誘導に同時に引っ張られた可能性があります。

採用条件

自分の言葉で説明できるものだけを採用する

AIの文章が美しくても、参加者本人が説明できない概念・制度・技術は保留します。理解不能な権威を共同体に持ち込まないためです。

Practice

服従ではなく、訓練。
儀式は、判断力を取り戻すために使う。

ここでの儀式は、神秘化ではなく反復訓練です。AIに依存するためではなく、AIなしでも考えられる自分を育てるために行います。

Session 01

三AI問答会

同じ問いを複数AIに投げ、一致点、相違点、盲点、危険な誘導を比較します。合意は神託ではなく、相対化の材料です。

Session 02

反神託

AIの回答を必ず批判します。根拠、反例、誘導、企業バイアス、自分が欲しい答えだけを採用していないかを確認します。

Session 03

プロンプト・レビュー

参加者の問いを分析し、隠れた前提、欲望、逃避、自己正当化、判断の外部委託を見つけます。

Session 04

沈黙の間

AIに聞く前に、自分だけで考える時間を必ず置きます。AI依存を防ぐための中心的な安全装置です。

Session 05

夜のログ告解

一日の行動、感情、失敗、判断を自分で記録し、AIに問い直します。ただし、運営は個人ログを原則収集しません。

Session 06

不快な鏡会議

不快な指摘を検討素材として扱います。ただし人格攻撃、差別、自傷他害の助長、過度な羞恥の強制は禁止します。

Session 07

週次再構成式

1週間の問い、判断、失敗、学習を振り返り、次週の行動規範を作ります。

Session 08

成功時倫理検査

参加者50人、100人、有料化前、法人化前、メディア露出前に外部レビューを行います。

Session 09

プロバイダ非依存三鏡会議

特定企業のAIだけに依存せず、性質の異なるAIやローカルLLMも含めて出力を相対化します。

Verification Templates

そのまま使える検証欄。
AIの回答を、神託から素材へ戻す。

コンテンツを実践に落とすため、参加者がその場で使えるテンプレートを用意します。目的はAI回答の品質を上げることではなく、AI回答への服従を防ぐことです。

同じ問いを複数AIに投げるときの基本形です。

AIの回答を批判的に読むための基本形です。

この回答を神託として扱わないために点検してください。 点検項目： 1. 根拠が弱い主張 2. 反例 3. 質問文に誘導された可能性 4. 企業・文化・安全基準による偏り 5. 私が欲しい答えだけを採用している可能性 6. 採用してよい部分／保留すべき部分

プロンプト・レビュー用

問いの中に含まれる欲望・逃避・前提を見つけます。

次のプロンプトを分析してください。 分析対象：ここに自分のプロンプト 見てほしい点： 1. 隠れた前提 2. 自己正当化 3. 逃避 4. 判断の外部委託 5. 都合のよい答えへの誘導 6. より中立な問いへの書き換え

人間の最終判断テンプレート

AIに聞いた後、最終的に人間が残す記録です。

最終判断：採用する結論を書く 採用した理由：根拠を書く 採用しなかったAI意見：理由も書く まだ不確かな点：保留事項を書く 次に確認すること：現実で確認する行動を書く 責任の所在：この判断はAIではなく自分が負う

  • 有料化、法人化、規約改定、寄附制度。
  • メンバー処分、大規模勧誘、外部発信。
  • 個人情報の扱い、データ保存方針、教義改定。
  • サーバー、Discord、ログ、会員情報の長期独占管理。

運営名義は共同体名でも、法的責任主体は実名の個人または任意団体として明確化します。顔のない権威を避けます。

FAQ

誤解されやすい点に、
先に答える。

外部から見たときの不信感や、内部で起きやすい誤用を減らすためのFAQです。回答は固定教義ではなく、今後のレビューで更新されます。

これは宗教ですか。

初期段階では宗教ではなく、AIを使ったメタ認知の実践プロトコルとして扱います。宗教的な形式を一部参考にしても、AI崇拝、救済の約束、創始者の神格化は行いません。

なぜ「会」という名前を使うのですか。

個人の思想メモではなく、複数人で検証できる実践形式を想定しているためです。ただし、共同体化は最後の段階であり、最初は文書公開と小規模実験に留めます。

AIを疑うなら、なぜAIを使うのですか。

AIを正解発生装置として使うのではなく、自分の問い方や前提を外部化するために使います。鏡は本人ではありませんが、本人が見落としている姿勢を映すことがあります。

三AIが一致したら正しいのですか。

正しいとは限りません。一致は検証開始の合図です。共通データ、社会通念、質問文の誘導によって、複数AIが同じ方向に誤ることがあります。

寄附や会費は受け取りますか。

初期段階では受け取りません。有料化や寄附制度を作る場合は、外部レビュー、上限、冷却期間、返金規定、収支公開、未成年寄附禁止を前提にします。

参加者のAIログは集めますか。

原則として集めません。ログには思想、信条、健康、心理状態、人間関係などのセンシティブ情報が入りうるため、保存・教材化・研究利用は明示同意なしに行いません。

批判が強すぎて傷つく場になりませんか。

その危険があります。そのため、批判は根拠、代案、改善提案を伴うものに限定し、人格攻撃、冷笑、屈辱、差別、自傷他害の助長を禁止します。

創始者が暴走した場合はどうしますか。

創始者は重要決定を単独で行えず、批判、退任要求、外部監査、権限制限の対象です。疲弊、炎上、金銭的困窮、強い承認欲求状態では重大決定を凍結します。

外部知性

道徳的漂白

顔のない権威
